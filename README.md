## 🔌 API & Endpoints

**API Documentada:** Open Finance Brasil (OFB)

**Quantidade de Endpoints:** Múltiplos (bancos participantes)

**Base URL:** `https://api.openfinance.br (ou sandbox)`

**Documentação Oficial:** https://openfinancebrasil.org.br/

**Cobertura de Endpoints:**
Consentimentos, recursos, dados de usuários, bancos, investimentos

**Autenticação:**
- OAuth 2.0 / Client Credentials (padrão Open Finance / bancos)
- Tokens gerenciados automaticamente pelo CLI
- Configuração via variáveis de ambiente ou arquivo de config

**Exemplo de uso:**
```bash
# Listar endpoints disponíveis
open-finance-br --help

# Exemplo de chamada (ajustar conforme CLI)
# open-finance-br account list
```

    38|# open-finance-br-cli
    39|
    40|CLI Python para consumir APIs padronizadas do Open Finance Brasil como
    41|consumidor. Ele usa apenas biblioteca padrao (`urllib`, `argparse`, `json`) e
    42|gera respostas JSON coloridas no terminal.
    43|
    44|## Specs usadas
    45|
    46|- Accounts v2.4.2: https://raw.githubusercontent.com/OpenBanking-Brasil/openapi/main/swagger-apis/accounts/2.4.2.yml
    47|- Consents v3.0.1: https://raw.githubusercontent.com/OpenBanking-Brasil/openapi/main/swagger-apis/consents/3.0.1.yml
    48|- Resources v3.0.0: https://raw.githubusercontent.com/OpenBanking-Brasil/openapi/main/swagger-apis/resources/3.0.0.yml
    49|- Participantes do diretorio: https://data.directory.openbankingbrasil.org.br/participants
    50|
    51|Os mocks retornam payloads com os campos obrigatorios dessas specs para permitir
    52|desenvolvimento local quando nao ha instituicao real configurada.

    55|

    63|## Configuracao
    64|
    65|```bash
    66|openfinance config set \
    67|  --base-url https://api.banco.com.br/open-banking/accounts/v2 \
    68|  --token-url https://auth.banco.com.br/oauth2/token \
    69|  --client-id "$CLIENT_ID" \
    70|  --client-secret "$CLIENT_SECRET" \
    71|  --certificate /path/cert.pem \
    72|  --certificate-key /path/key.pem
    73|```
    74|
    75|URLs por familia podem ser configuradas separadamente quando a instituicao
    76|publicar bases diferentes:
    77|
    78|```bash
    79|openfinance config set \
    80|  --accounts-url https://api.banco.com.br/open-banking/accounts/v2 \
    81|  --consents-url https://api.banco.com.br/open-banking/consents/v3 \
    82|  --resources-url https://api.banco.com.br/open-banking/resources/v3
    83|```
    84|
    85|Arquivos locais:
    86|
    87|```bash
    88|openfinance config path
    89|openfinance config show
    90|```
    91|
    92|Por padrao os arquivos ficam em `~/.openfinance/config.json` e
    93|`~/.openfinance/token_cache.json`. Para testes, use `OPENFINANCE_CONFIG` e
    94|`OPENFINANCE_TOKEN_CACHE`.
    95|
    96|## Token OAuth2
    97|
    98|```bash
    99|openfinance token --scope "openid accounts consents resources"
   100|openfinance token --force
   101|```
   102|
   103|Quando nenhuma instituicao real ou credencial OAuth esta configurada, o CLI
   104|retorna e cacheia um token mock. Em uso real, configure `token_url`, `client_id`
   105|e `client_secret`; o token e obtido por OAuth2 `client_credentials` com HTTP
   106|Basic Auth e corpo `grant_type=client_credentials`.
   107|
   108|## Contas
   109|
   110|```bash
   111|openfinance accounts
   112|openfinance accounts --page 1 --page-size 25 --account-type CONTA_DEPOSITO_A_VISTA
   113|openfinance account mock-account-001
   114|openfinance balances mock-account-001
   115|openfinance transactions mock-account-001 --from-booking-date 2026-01-01 --to-booking-date 2026-05-25
   116|openfinance transactions-current mock-account-001
   117|```

   119|## Consentimentos
   120|
   121|Criar consentimento com payload montado pelo CLI:
   122|
   123|```bash
   124|openfinance consents create \
   125|  --cpf 12345678901 \
   126|  --permissions ACCOUNTS_READ ACCOUNTS_BALANCES_READ ACCOUNTS_TRANSACTIONS_READ RESOURCES_READ \
   127|  --expiration-date-time 2026-12-31T23:59:59Z
   128|```
   129|
   130|Usar payload JSON proprio:
   131|
   132|```bash
   133|openfinance consents create --payload consent.json
   134|openfinance consents get urn:openfinancebrcli:example
   135|openfinance consents delete urn:openfinancebrcli:example
   136|```

   138|## Diretorio e recursos
   139|
