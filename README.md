    66|    38|# open-finance-br-cli
    67|    39|
    68|    40|CLI Python para consumir APIs padronizadas do Open Finance Brasil como
    69|    41|consumidor. Ele usa apenas biblioteca padrao (`urllib`, `argparse`, `json`) e
    70|    42|gera respostas JSON coloridas no terminal.
    71|    43|
    72|    44|## Specs usadas
    73|    45|
    74|    46|- Accounts v2.4.2: https://raw.githubusercontent.com/OpenBanking-Brasil/openapi/main/swagger-apis/accounts/2.4.2.yml
    75|    47|- Consents v3.0.1: https://raw.githubusercontent.com/OpenBanking-Brasil/openapi/main/swagger-apis/consents/3.0.1.yml
    76|    48|- Resources v3.0.0: https://raw.githubusercontent.com/OpenBanking-Brasil/openapi/main/swagger-apis/resources/3.0.0.yml
    77|    49|- Participantes do diretorio: https://data.directory.openbankingbrasil.org.br/participants
    78|    50|
    79|    51|Os mocks retornam payloads com os campos obrigatorios dessas specs para permitir
    80|    52|desenvolvimento local quando nao ha instituicao real configurada.

    91|    63|## Configuracao
    92|    64|
    93|    65|```bash
    94|    66|openfinance config set \
    95|    67|  --base-url https://api.banco.com.br/open-banking/accounts/v2 \
    96|    68|  --token-url https://auth.banco.com.br/oauth2/token \
    97|    69|  --client-id "$CLIENT_ID" \
    98|    70|  --client-secret "$CLIENT_SECRET" \
    99|    71|  --certificate /path/cert.pem \
   100|    72|  --certificate-key /path/key.pem
   101|    73|```
   102|    74|
   103|    75|URLs por familia podem ser configuradas separadamente quando a instituicao
   104|    76|publicar bases diferentes:
   105|    77|
   106|    78|```bash
   107|    79|openfinance config set \
   108|    80|  --accounts-url https://api.banco.com.br/open-banking/accounts/v2 \
   109|    81|  --consents-url https://api.banco.com.br/open-banking/consents/v3 \
   110|    82|  --resources-url https://api.banco.com.br/open-banking/resources/v3
   111|    83|```
   112|    84|
   113|    85|Arquivos locais:
   114|    86|
   115|    87|```bash
   116|    88|openfinance config path
   117|    89|openfinance config show
   118|    90|```
   119|    91|
   120|    92|Por padrao os arquivos ficam em `~/.openfinance/config.json` e
   121|    93|`~/.openfinance/token_cache.json`. Para testes, use `OPENFINANCE_CONFIG` e
   122|    94|`OPENFINANCE_TOKEN_CACHE`.

   124|    96|## Token OAuth2
   125|    97|
   126|    98|```bash
   127|    99|openfinance token --scope "openid accounts consents resources"
   128|   100|openfinance token --force
   129|   101|```
   130|   102|
   131|   103|Quando nenhuma instituicao real ou credencial OAuth esta configurada, o CLI
   132|   104|retorna e cacheia um token mock. Em uso real, configure `token_url`, `client_id`
   133|   105|e `client_secret`; o token e obtido por OAuth2 `client_credentials` com HTTP
   134|   106|Basic Auth e corpo `grant_type=client_credentials`.
   135|   107|

   147|   119|## Consentimentos
   148|   120|
   149|   121|Criar consentimento com payload montado pelo CLI:
   150|   122|
   151|   123|```bash
   152|   124|openfinance consents create \
   153|   125|  --cpf 12345678901 \
   154|   126|  --permissions ACCOUNTS_READ ACCOUNTS_BALANCES_READ ACCOUNTS_TRANSACTIONS_READ RESOURCES_READ \
   155|   127|  --expiration-date-time 2026-12-31T23:59:59Z
   156|   128|```
   157|   129|
   158|   130|Usar payload JSON proprio:
   159|   131|
   160|   132|```bash
   161|   133|openfinance consents create --payload consent.json
   162|   134|openfinance consents get urn:openfinancebrcli:example
   163|   135|openfinance consents delete urn:openfinancebrcli:example
   164|   136|```
